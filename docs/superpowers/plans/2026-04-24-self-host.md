# silva-omnium 자체호스팅 (M1 Mac mini + 웹 편집기) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** M1 Mac mini 8GB 를 silva-omnium 의 24/7 호스트로 만든다. 웹 브라우저에서 raw/wiki 파일을 트리·에디터·미리보기로 편집할 수 있고, 회사 M3 Max·Galaxy S23 등 어디서든 안전하게 접근 가능하다.

**Architecture:** brew + launchd 네이티브 스택 (Docker 없음 — 8GB RAM 보호). Caddy 가 단일 HTTPS 입구에서 `/edit/*` 는 code-server 로, `/` 는 Astro `dist/` 정적 서빙으로 라우팅. fswatch 데몬이 `raw/` 변경을 감지하면 debounce 후 `make ingest && make build` 를 자동 실행. Tailscale Funnel 이 `<host>.<tailnet>.ts.net` 공개 URL 을 무료 발급, 회사 노트북은 클라이언트 설치 없이 브라우저로 접근, 개인 디바이스(S23 등)는 Tailscale 앱으로 사설 IP 접근.

**Tech Stack:** macOS launchd, Caddy 2 (auto-HTTPS, JSON-light Caddyfile), code-server (Coder), Astro 6 정적 출력 (이미 동작), fswatch, Tailscale (Funnel)

각 파일의 책임:

- **infra/Caddyfile**: 단일 진입점 — `/edit/*` → `localhost:8080` (code-server), `/api/*` → 미래 확장, `/*` → `web/dist/` 정적

- **infra/launchd/\*.plist**: macOS 표준 launchd 서비스 정의. `launchctl load` 로 등록 → 부팅 시 자동 시작 + 죽으면 재시작

- **infra/tailscale-setup.md**: Tailscale 가입·머신 등록·Funnel `tailscale serve --bg https / proxy 443` 명령 단계별

## Task 2: infra/Caddyfile 작성 + 검증

**Files:**
- Create: `infra/Caddyfile`

Caddy 의 라우팅: 단일 호스트 (Tailscale Funnel 이 노출하는 hostname) 에서 path 별 분기.

- [ ] **Step 1: Caddyfile 작성**

```caddy
# silva-omnium reverse proxy
#
# 호스트네임은 install 시 환경변수 SILVA_HOST 로 치환되거나, Tailscale Funnel
# 의 자동 hostname 을 그대로 쓴다. localhost 는 로컬 테스트용.

{
    # Tailscale 이 TLS 종료를 처리하므로 Caddy 는 평문 HTTP 만.
    # Funnel 외부 접근 시: 외부 → Funnel(HTTPS) → localhost:80(HTTP)
    auto_https off
}

:80 {
    # code-server: VS Code in browser
    handle /edit/* {
        reverse_proxy localhost:8080
    }
    handle /edit {
        redir /edit/ permanent
    }

    # 정적 위키 (Astro build 산출물)
    handle {
        root * /Users/silva/silva-omnium/web/dist
        try_files {path} {path}/index.html /404.html
        file_server
    }

    # 운영 로그
    log {
        output file /var/log/silva-omnium/caddy-access.log
        format console
    }
}
```

> 참고: `/Users/silva/silva-omnium/web/dist` 는 install 단계에서 `${HOME}` + repo path 로 치환된다 (Caddy template 기능 또는 `envsubst` 사용 — Task 6 참조).

## Task 4: ingest-watcher.sh (fswatch debounce wrapper)

**Files:**
- Create: `infra/ingest-watcher.sh`

raw/ 또는 wiki/_meta/_schema.md 변경을 감지하면 debounce 후 `make ingest && make build`. fswatch 가 동일 변경에 여러 이벤트를 쏠 수 있으므로 sleep 기반 debounce 사용.

- [ ] **Step 1: 스크립트 작성**

```bash
#!/usr/bin/env bash
#
# silva-omnium ingest watcher
#
# raw/ 변경 시 디바운스 후 make ingest && make build 실행.
# launchd 가 호출하며 SILVA_REPO 환경변수를 주입한다.
#
set -euo pipefail

REPO="${SILVA_REPO:-${HOME}/silva-omnium}"
DEBOUNCE_SECONDS="${SILVA_DEBOUNCE:-5}"
LOG="${SILVA_LOG:-/var/log/silva-omnium/ingest-watcher.log}"

mkdir -p "$(dirname "$LOG")"

log() {
    printf '[%s] %s\n' "$(date -u +%FT%TZ)" "$*" >> "$LOG"
}

run_pipeline() {
    log "pipeline start"
    if (cd "$REPO" && make ingest && make build) >> "$LOG" 2>&1; then
        log "pipeline ok"
    else
        log "pipeline FAILED (exit $?)"
    fi
}

if ! command -v fswatch >/dev/null; then
    log "FATAL: fswatch not installed. brew install fswatch"
    exit 1
fi

log "watcher starting on $REPO/raw"

# fswatch 가 변경을 보내면 debounce_pid 를 기록하고, 새 변경이 들어오면
# 기존 sleep 을 죽인 뒤 새로 잠재운다. sleep 이 끝나야 run_pipeline 이 호출된다.
debounce_pid=""

fswatch -o "$REPO/raw" "$REPO/wiki/_meta" | while read -r _; do
    log "change detected"
    if [[ -n "$debounce_pid" ]] && kill -0 "$debounce_pid" 2>/dev/null; then
        kill "$debounce_pid" 2>/dev/null || true
    fi
    (
        sleep "$DEBOUNCE_SECONDS"
        run_pipeline
    ) &
    debounce_pid=$!
done
```

## Task 6: install-mac-mini.sh (1회 부트스트랩)

**Files:**
- Create: `infra/install-mac-mini.sh`

이 스크립트는 M1 mini 에서 한 번 실행한다. brew → 도구 일괄 → repo clone (이미 있으면 skip) → 비밀번호 생성 → plist 치환·복사·load → 마지막에 Tailscale 안내.

- [ ] **Step 1: 스크립트 작성**

```bash
#!/usr/bin/env bash
#
# silva-omnium M1 Mac mini 1회 부트스트랩.
#
# 전제: macOS Sonoma+, 사용자 셸 zsh, 인터넷 가능.
# 실행: bash infra/install-mac-mini.sh  (repo 루트에서)
#
set -euo pipefail

REPO="$(cd "$(dirname "$0")/.." && pwd)"
USER_NAME="$(id -un)"
HOME_DIR="$HOME"
LOG_DIR="/var/log/silva-omnium"

say() { printf '\n\033[1;32m==> %s\033[0m\n' "$*"; }
warn() { printf '\033[1;33mWARN: %s\033[0m\n' "$*" >&2; }
fatal() { printf '\033[1;31mFATAL: %s\033[0m\n' "$*" >&2; exit 1; }

# ---------- preflight ----------
say "1/8  preflight"
[[ "$(uname -s)" == "Darwin" ]] || fatal "macOS 만 지원"
[[ "$(uname -m)" == "arm64" ]] || warn "Apple Silicon 가정. Intel 에선 brew prefix 가 다름"
command -v brew >/dev/null || fatal "Homebrew 필요. https://brew.sh"

# ---------- brew packages ----------
say "2/8  brew install"
brew install caddy fswatch tailscale node@20 python@3.12 || true
brew install --cask code-server || brew install code-server || warn "code-server 설치 실패 — 수동 설치 필요"

# ---------- python venv (silva ingest) ----------
say "3/8  python venv + deps"
python3 -m venv "$REPO/.venv"
"$REPO/.venv/bin/python3" -m pip install --quiet --upgrade pip
"$REPO/.venv/bin/python3" -m pip install --quiet -r "$REPO/scripts/requirements.txt"

# ---------- node (web/) ----------
say "4/8  npm install (web)"
(cd "$REPO/web" && npm install --silent)
test -L "$REPO/web/src/content/docs" || (cd "$REPO/web/src/content" && ln -s ../../../wiki docs)

# ---------- log directory ----------
say "5/8  log directory"
sudo mkdir -p "$LOG_DIR"
sudo chown "$USER_NAME":staff "$LOG_DIR"

# ---------- secrets: code-server password ----------
say "6/8  code-server config (비밀번호 자동 생성)"
mkdir -p "$HOME_DIR/.config/code-server"
if [[ -f "$HOME_DIR/.config/code-server/config.yaml" ]]; then
    warn "$HOME_DIR/.config/code-server/config.yaml 이미 존재 — 보존"
else
    PASSWORD="$(openssl rand -base64 18 | tr -d '/+=' | head -c 24)"
    sed "s|PASSWORD_PLACEHOLDER|$PASSWORD|" \
        "$REPO/infra/code-server-config.yaml.example" \
        > "$HOME_DIR/.config/code-server/config.yaml"
    chmod 600 "$HOME_DIR/.config/code-server/config.yaml"
    say "    code-server 비밀번호 생성됨 → $HOME_DIR/.config/code-server/config.yaml"
    say "    >>> 비밀번호: $PASSWORD"
    say "    >>> 위 비밀번호를 비밀번호 매니저에 저장하세요."
fi

# ---------- Caddyfile.runtime ----------
say "7/8  Caddyfile.runtime 생성 (path 치환)"
sed "s|/Users/silva/silva-omnium|$REPO|g" \
    "$REPO/infra/Caddyfile" \
    > "$REPO/infra/Caddyfile.runtime"
caddy validate --config "$REPO/infra/Caddyfile.runtime" --adapter caddyfile

# ---------- launchd plists ----------
say "8/8  launchd 등록"
mkdir -p "$HOME_DIR/Library/LaunchAgents"
for src in "$REPO"/infra/launchd/*.plist; do
    name="$(basename "$src")"
    dst="$HOME_DIR/Library/LaunchAgents/$name"
    sed -e "s|__USER__|$USER_NAME|g" \
        -e "s|__HOME__|$HOME_DIR|g" \
        -e "s|__REPO__|$REPO|g" \
        "$src" > "$dst"
    plutil -lint "$dst" >/dev/null
    launchctl unload "$dst" 2>/dev/null || true
    launchctl load "$dst"
    say "    loaded: $name"
done

# ---------- 마무리 안내 ----------
cat <<'NEXT'

==> 다음 단계 (수동):

1. 한 번 ingest + build 가 돌도록 raw/ 폴더에 첫 노트를 만들거나,
   현재 wiki 만으로 build 를 즉시 돌리려면:

      cd "$REPO" && make build

2. Tailscale 설정:

      open -a Tailscale     # GUI 로그인 (구글/깃허브)
      sudo tailscale up

3. Funnel 활성화 (Caddy 가 :80 에서 듣고 있다고 가정):

      sudo tailscale serve --bg --https=443 --set-path / http://localhost:80

   완료되면 다음 URL 이 출력됨:
      https://<machine>.<tailnet>.ts.net

4. 같은 URL 의 /edit/ 로 들어가면 code-server 비밀번호 입력 화면.
   루트 /  는 위키.

5. 회사 노트북·Galaxy S23·외부 디바이스 모두 위 URL 로 접근 가능.
   개인 디바이스는 Tailscale 앱 로그인 후 사설 IP 로도 접근 가능.

NEXT
```

## Task 7: infra/tailscale-setup.md (수동 단계 안내)

**Files:**
- Create: `infra/tailscale-setup.md`

- [ ] **Step 1: 문서 작성**

````markdown
# Tailscale + Funnel 설정 가이드

`install-mac-mini.sh` 가 brew 로 tailscale 을 설치한 뒤 이 단계를 수동으로 한다.

## 1. 가입 + 머신 등록

```bash
# 트레이 GUI 로 로그인 (Google / GitHub / 이메일)
open -a Tailscale

# CLI 로 머신 등록 (브라우저 열림 → 승인)
sudo tailscale up
```

확인:

```bash
tailscale status
# 머신 이름과 100.x.x.x IP 가 출력되면 OK
```

## 2. Funnel 활성화 (필요 시 admin 콘솔에서 권한 켜기)

기본은 비활성. https://login.tailscale.com/admin/dns 의 "Funnel" 섹션에서 본인 tailnet 활성화.

## 3. Caddy(:80) 를 Funnel(:443) 로 노출

```bash
sudo tailscale serve --bg --https=443 --set-path / http://localhost:80
```

출력에 다음과 같은 URL 표시:

```
Available within your tailnet:
  https://<host>.<tailnet>.ts.net/

Available on the public internet:
  https://<host>.<tailnet>.ts.net/    (Funnel)
```

## 4. 검증

- 같은 머신 브라우저: `http://localhost/` → wiki 보임
- 같은 머신 브라우저: `http://localhost/edit/` → code-server 로그인 화면
- 다른 디바이스(Tailscale 안): 위 ts.net URL 로 동일 결과
- 회사 노트북(Tailscale 없음): 위 ts.net URL → public Funnel 통해 접근 → code-server 비밀번호 필요

## 5. 끄기 / 변경

```bash
sudo tailscale serve status            # 현재 노출 상태
sudo tailscale serve reset             # 전체 해제
sudo tailscale funnel --bg off         # Funnel 만 끄기 (사설 접근은 유지)
```

## 트러블슈팅

- **502 Bad Gateway**: Caddy 가 :80 에 안 떠 있다. `launchctl list | grep silva-omnium` 으로 확인. `tail -f /var/log/silva-omnium/caddy.err`
- **/edit 만 안 됨**: code-server 가 :8080 에 안 떠 있다. `tail -f /var/log/silva-omnium/code-server.err`
- **로그인 후 흰 화면**: code-server base path. Caddy 의 reverse_proxy 가 `/edit/*` 의 prefix 를 그대로 전달하는지 확인. 필요 시 Caddyfile 에 `handle_path /edit/*` 로 변경 (prefix 제거).
- **Funnel URL 이 Cloudflare 같은 곳 거치는 느낌**: Funnel 은 Tailscale 자체 엣지를 통과 — 첫 번째 접속만 약간 느릴 수 있음, 이후 캐시.
````
